# BLUEPRINT 6: NQ Situation Room — SitDeck-style war-room page wrapping SweepStats

**BUILDER:** Claude Sonnet, working alone, cold start, cannot ask questions.
(Single-file HTML generation from known JSONs, but the design bar is the
product — Sonnet, and it must follow the design spec literally.)

## GOAL

One command, `python situation_room.py`, renders `results/situation_room.html`:
a self-contained, dark, spy-terminal "NQ Situation Room" page (inspired by
sitdeck.com's OSINT dashboard aesthetic) that assembles everything this repo
already generates — live-ish NQ tape, GEX level ladder, SweepStats fakeout
statistics, London-manipulation bias, session clocks, sectors, calendar,
news — into one fixed-layout screen that looks screenshot-worthy enough to
post. `--png` also renders `results/situation_room.png` via headless Chrome.
This is the product WRAPPER for SweepStats, not a new platform: no server,
no accounts, no drag-and-drop, no new data sources in v1.

Why it exists (2026-07-08 decision): SitDeck went viral on looks alone
(2M+ views, ~7k signups <24h). The commodity part (widget grid) is not the
play; the play is proprietary SweepStats/GEX stats wearing that aesthetic —
it upgrades the daily stats-card posting loop with a far more shareable
artifact and is the candidate landing face for the future product.

## CONTEXT THE BUILDER NEEDS

- Files to read first: `backtest/stats_card.py` (how this repo builds
  self-contained HTML from the summary JSON — reuse its aggregation idioms,
  NOT its visual theme), `backtest/daily_stats_run.py` (`_find_chrome` +
  headless PNG render pattern — copy it), `CLAUDE.md` (repo hygiene).
- Data inputs, all already regenerated by existing pipelines (read only):
  - `results/morning_brief.json` — `{"generated": iso, "ctx": {NQ|ES|SPY|QQQ|VIX: {prev_close,last,chg_pct}}, "sectors": {XLK|XLF|XLE|XLV|IWM|GLD|TLT|DX-Y.NYB: {name,last,chg_pct}}, "calendar": [...], "news": [{title,date,url},...], "ibkr": {...}, "positions": [...]}`
  - `results/gex_levels.json` — `{"timestamp": iso, "symbol", "current_price", "levels": [{"name","price"},...]}`. GOTCHA: can be weeks stale (held June 24 on July 7) — trust the timestamp field.
  - `results/session_stats_summary.json` — `{"sample": {sessions,from,to}, "london_manipulation": {asia_high|asia_low|none|both: {days,ny_up,ny_up_pct,median_ny_change_pts}}, "first_touch": {asia_high|asia_low|london_high|london_low: {touched,touched_pct,fakeout,break,fakeout_pct,fakeout_median_mfe60_pts,fakeout_median_mae120_pts,...}}, "time_buckets": {...}, "generated": iso}`
  - Hero stat = sum(fakeout)/sum(touched) across `first_touch` values (see
    `build_card()` in stats_card.py — 61% on 47 sessions as of 2026-07-08).
- Any input file may be missing or stale. The page must still render: a
  missing file shows the panel with "NO FEED"; a stale file (source
  timestamp older than 24h) renders the panel dimmed with an amber
  "STALE · <n>h AGO" chip. Every panel shows its data age.
- News item `date` is RFC 2822 (e.g. "Tue, 07 Jul 2026 12:03:21 +0000") —
  parse with `email.utils.parsedate_to_datetime`.

## CONSTRAINTS

- Must stay inside: new file `situation_room.py` (repo root, next to
  `morning_brief.py`), two `.gitignore` lines
  (`results/situation_room.html`, `results/situation_room.png`), one new
  CLAUDE.md subsection (step 5 of the blueprint plan).
- Must not change: `morning_brief.py`, `backtest/*`, `data/*`, the
  startup-protocol wording in CLAUDE.md, any existing output file.
- Stack: Python 3.14 stdlib only (json, datetime, zoneinfo, email.utils,
  argparse, subprocess, webbrowser). All CSS/JS inline — the HTML file must
  work offline with zero external requests (fonts included: use local
  stacks, no webfont downloads).
- Non-negotiables: read-only against the input JSONs; never crashes on a
  missing/stale/malformed input (degrade per-panel); footer must carry
  "Past frequency ≠ future probability. Not financial advice."

## STEP-BY-STEP PLAN

1. Create `situation_room.py` with `load()` helpers returning
   `(data|None, age_hours|None)` per input file, and `main()` with argparse:
   `--png` (render PNG after HTML), `--open` (open HTML in browser).
2. Build the page as one fixed CSS grid (no drag-drop), 1440px design width,
   laid out as:
   - **Header bar**: "NQ SITUATION ROOM" wordmark left (mono, letterspaced),
     center = live UTC + ET clocks (JS, ticking), right = "SWEEPSTATS DATA ·
     n=<sessions> SESSIONS" chip + build timestamp.
   - **Row 1 — tape strip**: NQ big (last, chg_pct, up/down colored), then
     ES / SPY / QQQ / VIX compact, then the 8 sector chips inline.
   - **Row 2, left (2/4 width) — GEX LEVEL LADDER**: vertical price ladder of
     `gex_levels.levels` sorted by price desc, current_price row highlighted,
     wall/flip names emphasized, `[+]`/`[++]`/`[+++]` markers rendered as
     amber intensity dots. Stale chip rules apply (this feed is the usual
     offender).
   - **Row 2, mid (1/4) — SESSION CLOCK**: Asia 18:00–00:00, London
     02:00–05:00, NY 09:30–16:00 (all ET) as three status rows with
     OPEN (green) / CLOSED (dim) computed live in JS from the visitor's
     clock converted to ET (use `Intl.DateTimeFormat` with
     timeZone "America/New_York" — do NOT hand-roll offsets), plus
     countdown to the next session boundary.
   - **Row 2, right (1/4) — HERO STAT**: giant fakeout % + "of first NY
     touches of Asia/London levels are fakeouts" + n=<touches> episodes,
     <sessions> sessions <from>→<to>.
   - **Row 3, left (1/2) — FIRST-TOUCH BOARD**: 4 rows (Asia H/L, London
     H/L): fakeout% bar, touched count, median reversal pts/60m, stop-run
     risk pts (`fakeout_median_mae120_pts`).
   - **Row 3, right (1/2) — LONDON MANIPULATION MATRIX**: the 4 sweep
     patterns with days, NY-up %, median NY move (green/red), "(low sample)"
     tag when days < 5.
   - **Row 4 — WIRE**: latest 6 news items as one-line entries
     `HH:MM · title` each a link (target="_blank"; repo rule: headlines are
     always clickable), plus today's calendar events if any.
   - **Footer**: data-source ages line ("BRIEF 3h · GEX 14d STALE ·
     STATS 0h"), the not-financial-advice line, and "SweepStats — NQ
     session-level statistics" as the product sign-off.
3. Visual spec — follow LITERALLY, do not substitute the repo dashboard
   theme (this page is a product brand artifact, deliberately distinct):
   - Palette: page `#0a0d10`, panel `#10151b`, panel border `#1e2833`,
     text `#c9d4de`, dim `#5c6a78`, amber accent `#ffb454`, alert red
     `#ff5f56`, green `#3ddc84`, cyan detail `#4dd0e1`.
   - Type: headers + numbers in `"Cascadia Mono", "Consolas", monospace`;
     body in `"Segoe UI", system-ui, sans-serif`. Panel titles 11px,
     uppercase, letter-spacing .12em, dim, with a small amber `▮` prefix.
   - Texture: 1px panel borders, faint background grid
     (`repeating-linear-gradient`, 24px, rgba(255,255,255,.015)), subtle
     top scanline strip on the header. No border-radius > 4px, no
     gradients on data, no purple.
   - DECISION: fixed dark only, no light theme — war-room artifact, and
     screenshots must look identical for everyone.
4. PNG render: copy `daily_stats_run.py`'s chrome discovery + subprocess
   call, `--window-size=1440,1600`, output `results/situation_room.png`;
   skip with a warning if Chrome is missing.
5. Add to CLAUDE.md, after the TiltGuard section, a short "## Situation
   Room" subsection: the command, the `--png`/`--open` flags, that it's
   read-only over `results/*.json` + regenerates whenever asked ("open the
   situation room" / "update the war room"), and that panels dim as STALE
   rather than hiding when a feed is old.
6. Add the two `.gitignore` lines (regenerated outputs, per repo hygiene).
7. Test matrix (run all):
   a. `python situation_room.py` with the real results/ files → exits 0,
      HTML exists, contains the hero % and "STALE" chip on the GEX panel
      if gex_levels.json is older than 24h.
   b. Rename `results/morning_brief.json` temporarily → rerun → page still
      renders with "NO FEED" tape panel → restore the file.
   c. `python situation_room.py --png` → PNG exists and is > 30KB.
   d. Read the PNG visually and check it against the visual spec (step 3)
      before calling it done.
